# ROI Analyzer 계산식 문서

## 1. 채널별 지표 (calcChannelMetrics)

| 지표 | 계산식 | 설명 |
|------|--------|------|
| **매출원가(COGS)** | `단가 × 판매수량` | 상품 원가 총합 |
| **수수료** | `매출 × 수수료율` | 쿠팡 기본 10.8% |
| **영업이익** | `매출 - 매출원가 - 광고비 - 수수료 - 물류비 - 반품비 - 기타비용` | 순수 영업 수익 |
| **영업이익률** | `(영업이익 / 매출) × 100` | % 단위 |
| **ROAS** | `(매출 / 광고비) × 100` | 광고 수익률, % 단위 |

## 2. 상품 종합 지표 (calcProductMetrics)

### 2.1 기본 합산
| 지표 | 계산식 |
|------|--------|
| **총 매출** | 모든 채널 매출 합산 |
| **총 광고비** | 모든 채널 광고비 합산 |
| **총 수수료** | 모든 채널 수수료 합산 |
| **총 물류비** | 모든 채널 물류비 합산 |
| **총 반품비** | 모든 채널 반품비 합산 |
| **총 매출원가** | 모든 채널 COGS 합산 |
| **총 기타비용** | 모든 채널 기타비용 합산 |
| **총 판매수량** | 모든 채널 판매수량 합산 |
| **총 리뷰수** | 모든 채널 리뷰수 합산 |

### 2.2 수익성 지표
| 지표 | 계산식 | 설명 |
|------|--------|------|
| **총 영업이익** | `총매출 - 총매출원가 - 총광고비 - 총수수료 - 총물류비 - 총반품비 - 총기타비용` | |
| **영업이익률** | `(총영업이익 / 총매출) × 100` | 15% 이상 양호 |
| **통합 ROAS** | `(총매출 / 총광고비) × 100` | 200% 이상 손익분기 |

### 2.3 투자 수익 지표
| 지표 | 계산식 | 설명 |
|------|--------|------|
| **투자금** | `수입수량 × (매입단가 + 물류비)` | 또는 수동 입력값 |
| **기간 ROI** | `(총영업이익 / 투자금) × 100` | 분석기간 내 수익률 |
| **연환산 ROI** | `기간ROI × (365 / 분석기간일수)` | 연간 기준 환산 |
| **투자점수** | `min(100, max(0, round((연환산ROI / 160) × 100)))` | 0~100점 스케일 |

### 2.4 회전율/효율 지표
| 지표 | 계산식 | 설명 |
|------|--------|------|
| **일평균 판매량** | `총판매수량 / 분석기간일수` | 일 단위 |
| **재고 소진 예상일** | `수입수량 / 일평균판매량` | 현 속도 유지 시 |
| **연간 회전율** | `365 / 재고소진예상일` | 1년에 몇 회전 |
| **일평균 이익** | `총영업이익 / 분석기간일수` | |
| **투자금 회수일** | `투자금 / 일평균이익` | 이익이 양수일 때만 |
| **자본 회전율** | `총매출 / 투자금` | |

### 2.5 품질/고객 지표
| 지표 | 계산식 | 설명 |
|------|--------|------|
| **가중 반품률** | `Σ(채널반품률 × 채널판매수량) / 총판매수량` | 판매량 가중 평균 |
| **가중 평점** | `Σ(채널평점 × 채널리뷰수) / 총리뷰수` | 리뷰수 가중 평균 |
| **가중 전환율** | `Σ(채널CVR × 채널판매수량) / 총판매수량` | |

### 2.6 손익분기 ROAS (BE-ROAS)
| 지표 | 계산식 | 설명 |
|------|--------|------|
| **평균 마진율** | `(총매출 - 총매출원가) / 총매출` | |
| **평균 수수료율** | `총수수료 / 총매출` | |
| **BE-ROAS** | `1 / (평균마진율 - 평균수수료율) × 100` | 이 ROAS 이상이면 흑자 |

## 3. 상품 상태 판정

| 상태 | 조건 | 의미 |
|------|------|------|
| **확대** | 투자점수 ≥ 75 AND 연환산ROI ≥ 100% AND 영업이익률 ≥ 15% | 투자 확대 권장 |
| **유지** | 위 조건 미충족 AND 영업이익 > 0 | 현 수준 유지 |
| **중단(stop)** | 영업이익 < 0 AND 모든 주간 추이가 적자 | 즉시 중단 검토 |
| **축소** | 그 외 | 투자 축소 검토 |

## 4. 마케팅 효율 지표

| 지표 | 계산식 | 설명 |
|------|--------|------|
| **카테고리별 ROAS** | `(기대매출 / 지출금액) × 100` | 마케팅 채널별 |
| **카테고리별 ROI** | `((기대매출 - 지출금액) / 지출금액) × 100` | |
| **이익 기여도** | `기대매출 - 지출금액` | 순이익 기여분 |
| **비용 비중** | `(채널지출 / 전체마케팅비) × 100` | |

## 5. 대시보드 건강성 점수 (healthScore)

| 항목 | 배점 | 조건 |
|------|------|------|
| **수익성** | 25점 | 총이익 > 0 |
| **ROI** | 20점 | 평균ROI > 50%: 20점, > 0%: 10점 |
| **ROAS** | 20점 | 평균ROAS > 200%: 20점, > 100%: 10점 |
| **반품률** | 20점 | 평균반품률 < 5%: 20점, < 10%: 10점 |
| **안정성** | 15점 | 위험상품 < 10%: 15점, < 30%: 8점 |
| **합계** | **100점** | |

## 6. 예측 분석 (ForecastPage)

| 지표 | 계산식 | 설명 |
|------|--------|------|
| **일평균 매출** | `총매출 / 분석기간일수` | |
| **성장률** | `((최근일평균 - 이전일평균) / 이전일평균) × 100` | 추세 기반 |
| **3개월 예상 매출** | `일평균매출 × (1 + 성장률/100) × 90` | |
| **6개월 예상 매출** | `일평균매출 × (1 + 성장률/100) × 180` | |
| **3개월 예상 이익** | `3개월예상매출 × (영업이익률 / 100)` | |
| **투자 효율** | `연환산ROI × 연간회전율` | 종합 효율 점수 |
| **리스크 점수** | 반품률(30) + ROI(40) + ROAS(30) | 0~100, 높을수록 위험 |

### 리스크 점수 세부
| 항목 | 점수 |
|------|------|
| 반품률 > 10% | +30 |
| 반품률 > 5% | +15 |
| 연환산ROI < 0% | +40 |
| 연환산ROI < 30% | +20 |
| ROAS < 100% | +30 |
| ROAS < 200% | +10 |

## 7. What-If 시뮬레이션

| 시뮬레이션 변수 | 적용 방식 |
|----------------|-----------|
| **광고비 증감(%)** | `새광고비 = 기존광고비 × (1 + 증감/100)` |
| **반품률 증감(%)** | `새반품률 = 기존반품률 + 증감값` |
| **판매가 증감(%)** | `새매출 = 기존매출 × (1 + 증감/100)` |
| **목표 ROAS 설정** | 목표 ROAS 달성에 필요한 광고비 역산 |

## 9. 엑셀 AI 분석 (CSV 매핑)

### 처리 흐름
```
1. CSV 업로드
2. 저장된 매핑 규칙 확인 (headerMatch 키워드로)
3-A. 규칙 있음 → 즉시 파싱
3-B. 규칙 없음 → Claude AI가 헤더 분석 → 매핑 규칙 생성 → localStorage 저장
4. 파싱 결과: 매출, 수량, 환불 합산
5. 상품명 키워드 매칭 (2글자+ 키워드 3개+ 일치 시 동일 상품 판정)
6. 분석기간 날짜 필터링
7. 결과를 채널(기타몰)에 자동 반영
```

### 매핑 규칙 구조
```json
{
  "platform": "셀파이",
  "headerMatch": ["일자", "주문수"],
  "mapping": {
    "date": ["일자"],
    "revenue": ["순매출", "판매합계"],
    "qty": ["항목수", "판매수량"],
    "refund": ["환불합계"]
  },
  "revenueField": "revenue",
  "qtyField": "qty"
}
```
